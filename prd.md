# PRD: IP回線・携帯回線 統合管理システム（LineHub）

**更新日：** 2026年6月3日

## 1. 背景・目的

### 1.1 背景

現在、IP回線・携帯回線の再販事業においてスプレッドシートで以下を管理している。

- アジャストワン社から調達したIP回線契約（原価）
- 各テナント（顧客企業）への回線・チャンネル・電話番号の割り当て
- テナントごとの請求ルールに基づく請求額の確認・差分チェック
- SoftBankから受領する携帯回線の利用料超過代金の集計・Salesforce登録

スプレッドシート管理では以下の課題がある。

- 請求漏れ・過剰請求の検知が属人的・手作業
- 複数担当者が同時編集するとデータ整合性が壊れやすい
- 請求状況のサマリー・可視化が困難
- 携帯回線の超過代金集計・Salesforce登録が手動で工数がかかる

### 1.2 目的

IP回線・携帯回線の**原価（調達コスト）と売上（テナントへの請求）を一元管理**し、請求差分の自動検知・可視化・Salesforce自動登録ができるシステムを構築する。

## 5. 取引先管理仕様（旧：テナント管理仕様）

### 取引先コード（内部：slug）

- SF商談IDを自動使用（例：`006Q900001aE5U2IAK`）
- URLの `/Opportunity/〇〇/view` の〇〇部分（18桁CaseSafeID）
- 手動登録・CSV一括登録ともに共通ルール
- UI表示名は「取引先コード」。DB・API内では `slug` のまま

### CSV一括登録

```
会社名,SF商談ID,MFパートナーID,備考
株式会社サンプル,006Q900001aE5U2IAK,,
```

- 会社名・SF商談IDは必須
- SF商談ID重複時はスキップ

## 6. 機能要件

### 6.1 IP回線管理

（Phase 1より変更なし）

### 6.2 携帯回線管理

#### 6.2.1 SoftBankファイルインポート

- 対応形式：.xlsx / .csv（拡張子で自動判別）
- 対象：全社分まとめた1ファイル
- ヘッダー行：1行目=項目名、2行目=税区分、3行目からデータ
- 15種の超過項目を自動集計し、テナントごとに合算

#### 6.2.2 回線マスタ管理

| 機能 | 内容 |
|---|---|
| 個別登録 | フォームから1件ずつ登録 |
| CSV一括登録 | CSVファイルで複数件一括登録（テンプレートダウンロード付き） |
| 編集・削除 | 各行の鉛筆アイコンから編集 |
| 端末回収管理 | 未回収 / 回収済 / 回収不要 の3択で管理 |

**CSVフォーマット：**
```
電話番号,会社名,ステータス,契約開始日,解約日,備考
090-1234-5678,株式会社サンプル,契約中,2024-01-01,,
```

#### 6.2.3 契約端末一覧（`/mobile/devices`）

| 機能 | 内容 |
|---|---|
| 月ナビゲーション | 右上に `< 年月 >` 形式で配置 |
| 契約期間表示 | 契約開始日・終了日列（終了日入力済みはアンバー色強調） |
| 端末回収表示 | 未回収 / 回収済 / 回収不要 のバッジ表示 |
| 回収フィルター | すべて / 未回収のみ / 回収済のみ / 回収不要のみ |

#### 6.2.4 Salesforce連携

| フィールド | 値 |
|---|---|
| 商品コード | `CMP_OP_09`（固定） |
| 商品名 | `携帯_利用料超過代金`（固定） |
| 販売価格 | 超過分合計金額 |
| 数量 | `1`（固定） |
| 請求開始日 | 利用月+2ヶ月の1日 |
| 請求終了日 | 利用月+2ヶ月の末日 |
| 商談 | `tenants.sfOpportunityId` |

#### 6.4.1 CSV課金項目マスタ管理（インポート改善）✅ 完了

**背景:** 現在は15品目固定で超過代金を集計しているが、SoftBankが新規課金項目を追加した場合に対応できない。

**実装済み要件:**

| # | 要件 | 状態 |
|---|------|------|
| 1 | CSVに含まれる全項目（課金・非課金問わず）をDBに保持し、管理画面でタブ切替（課金 / 非課金）で一覧表示できる | ✅ |
| 2 | 列番号ではなくCSVヘッダの**項目名**でマッチング（列位置変動に対応） | ✅ |
| 3 | SoftBankインポート実行時、まずヘッダー1行目のみをスキャンして課金項目一覧をプレビュー表示。ユーザーが確認してからインポートを実行 | ✅ |
| 4 | マスタ未登録の新規項目が検出された場合、同一ポップアップ内で「課金」「非課金」「継続取込」「今回のみ」を選択してからDBに登録 | ✅ |
| 5 | 管理画面（`/mobile/billing-items`）から項目の手動追加・削除・設定変更ができる | ✅ |
| 6 | 管理画面の「CSVから取込」ボタンでSoftBank CSVのヘッダー1行目を読み込み、未登録項目を一括登録できる。登録済み項目はスキップ、各未登録項目に「課金／非課金／取り込まない」を選択可 | ✅ |

**新規テーブル: `mobile_billing_items`（課金項目マスタ）** ✅ 実装済み

```
mobile_billing_items
  id                -- UUID
  item_name         -- SoftBank CSV上の項目名（ユニークキー・名前マッチング）
  is_billable       -- true: 課金 / false: 非課金
  continuous_import -- true: 継続取込 / false: 今回のみ
  created_at
  updated_at
```

**SoftBankインポートフロー（確定版）:**

```
インポート実行
  ↓
[プレビューAPI] ヘッダー1行目のみスキャン（DB書込なし）
  ↓
確認ダイアログ表示
  ├─ 課金項目一覧（マスタ登録済み・課金）
  └─ 未登録項目（課金 / 非課金 / 継続取込 / 今回のみ を選択）
  ↓
「確認してインポート」クリック
  ↓
[インポートAPI] 実際のデータ集計・DB保存・未登録項目をマスタ登録
```

```

  id, tenant_id, phone_number, status,
```
